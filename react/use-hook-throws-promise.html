How use() Suspends by Throwing a Promise

Click Fetch User to watch exactly what happens when a component calls use(promise). React renders your component, use() checks the promise's status — if it's still pending, it literally throws the promise. The nearest <Suspense> boundary catches it, shows a fallback, and re-renders the component once the promise resolves.

// What use() does internally (simplified):
function use(promise) {
  // Check if the promise has already resolved
  if (promise.status === 'fulfilled') {
    // Promise already resolved — return the value synchronously
    return promise.value;
  }

  if (promise.status === 'rejected') {
    throw promise.reason; // Let error boundary handle it
  }

  // Promise is still pending...
  // Tag the promise so we can read it next time
  promise.then(
    value  => { promise.status = 'fulfilled'; promise.value = value; },
    reason => { promise.status = 'rejected';  promise.reason = reason; }
  );

  // 🔥 THIS IS THE KEY PART: throw the promise itself!
  // React's renderer catches this in a try/catch.
  // It knows "oh, this component isn't ready yet"
  // and shows the nearest <Suspense> fallback instead.
  throw promise;
}

// Your component — looks synchronous!
function UserProfile({ userPromise }) {
  const user = use(userPromise); // suspends here if pending
  return <h1>{user.name}</h1>;
}

// Usage with Suspense boundary:
<Suspense fallback={<Spinner />}>
  <UserProfile userPromise={fetchUser(id)} />
</Suspense>
Why throwing a promise works 
// React's renderer does something like this (very simplified):
function renderComponent(Component, props) {
  try {
    const elements = Component(props);
    // Component returned JSX — render it normally
    commitToDOM(elements);
  } catch (thrown) {
    if (thrown instanceof Promise) {
      // It's a promise! This component is "suspended".
      // 1. Show the nearest Suspense fallback
      showFallback(nearestSuspenseBoundary);

      // 2. Wait for the promise to settle
      thrown.then(() => {
        // 3. Re-render the component (use() will return the value this time)
        renderComponent(Component, props);
      });
    } else {
      // It's a real error — let error boundaries handle it
      throw thrown;
    }
  }
}
What about rejected promises?

If the promise rejects instead of resolving, use() throws promise.reason (the error) instead of the promise itself. Since this is a regular error (not a promise), React does not treat it as a Suspense signal — it bubbles up to the nearest Error Boundary (componentDidCatch / getDerivedStateFromError), which can render a fallback error UI.

In short: pending → throw promise (Suspense catches it), rejected → throw error (Error Boundary catches it), fulfilled → return value (component renders normally).
